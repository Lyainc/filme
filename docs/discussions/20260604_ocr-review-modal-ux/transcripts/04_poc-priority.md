# TOPIC 4 — PoC 검증 우선순위 (실행 결론)

## 발언
- **[LLM/Vision Expert]**: #34에서 박스 정확도 정량 측정부터. 실제 티켓 5~10장(CGV·롯데·메가박스 혼합)으로 "박스 중심점이 올바른 텍스트 위에 있는 비율" 측정. 70% 미만이면 박스 탭은 보조로도 미덥지 않음 → 순수 칩 후퇴.
- **[Frontend Engineer]**: 그 측정은 UI 없이 가능. `/api/ocr-boxes` 응답을 원본 이미지에 박스만 그리는 디버그 페이지면 충분. 모달 UI(#35) 만들기 전에 게이트로.
- **[Product Strategy Expert]**: 칩+드롭다운 검수 UI는 박스 정확도와 무관하게 진행 가능(텍스트+분류만 의존). 박스 측정 기다릴 필요 없음. 기반(주)이니 먼저 서야 함.
- **[UX Expert]**: 박스 측정 통과 시 오버레이+탭 연동 얹음. 실패 시 칩만으로 출시·박스 보류. 어느 쪽이든 제품 완성.
- **[Critical Practitioner]**: 단일 실패점 없는 순서. 동의.

## 결론 (전원 합의)
1. **#34 = 박스 정확도 정량 게이트** — 디버그 오버레이로 5~10장 측정, 중심점 적중률 ≥70% 기준.
2. **칩+드롭다운 검수 UI 병행** — 박스와 독립(텍스트+분류만), 기반이라 먼저.
3. 게이트 통과 시 박스 오버레이+탭을 #35에 보조로 추가. 실패 시 칩만 출시·박스 보류.
