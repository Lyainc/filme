# TOPIC 3 — 인터랙션 방향: 박스 탭 vs 칩 vs 하이브리드 (핵심)

쟁점: 7(근본 인터랙션 방향) + 3(재인식)

## Independent Statements
- **[Optimistic Practitioner]**: 사용자가 원하는 박스 탭을 살리고 싶음. 이미지 위에서 직접 줍는 직관성은 칩 목록이 못 줌.
- **[Critical Practitioner]**: 박스 탭은 좌표 정확도(T1)+터치 정밀도(T2) 두 리스크에 동시 종속. 칩 목록은 둘 다 우회. 추출 텍스트를 칩으로 나열+드롭다운 할당이면 좌표 틀려도 매핑 정확.
- **[LLM/Vision Expert]**: 텍스트 추출·의미 분류는 LLM 강점, 좌표만 약점. 칩(텍스트+자동분류)은 강점 위에, 박스 좌표는 약점 위에 섬. 신뢰도 기반이 다름.
- **[UX Expert]**: 둘은 배타적이지 않음. 이미지+박스는 "맥락 표시"로 보여주고(어디서 왔는지 — 시각 매력+검수 신뢰), 매핑 조작은 칩/드롭다운. 박스 탭하면 칩 하이라이트 연동. 박스 탭 성공 시 보너스, 실패해도 칩으로 처리 → 안 무너짐.
- **[Product Strategy Expert]**: 사용자 선호도 살리고 리스크도 줄임. "탭의 즐거움"은 박스 보여주고 탭 반응 주는 것만으로 상당 충족. 매핑 정확성을 좌표에 안 걸 뿐.
- **[Frontend Engineer]**: 박스 오버레이(시각)와 칩 리스트(조작) 분리하면 박스 정확도 나빠도 칩 경로 독립 동작 → 단일 실패점 제거.

## Q&A
- **[Moderator]**: 순수 칩 vs 하이브리드?
- **[Critical Practitioner]**: 순수 칩이 가장 안전하나 사용자 선호·이미지 맥락 가치 버림. 하이브리드가 나음 — 단 칩이 주(主), 박스 탭이 보조(補) 위계 분명해야. 박스 탭을 주로 두면 다시 좌표에 생명 걸게 됨.
- **[UX Expert]**: 위계가 핵심. 박스 탭=가속기, 칩+드롭다운=항상 작동하는 기반. 이 순서면 합의.

## Dialectic
- **Thesis**: 박스 탭이 사용자가 원하는 경험.
- **Antithesis**: 박스 탭은 좌표·터치 두 리스크 종속, 단독 취약.
- **Synthesis**: 하이브리드 — 칩/드롭다운 기반(주), 이미지+박스는 맥락·보조 가속기(補). 탭 성공 시 빠른 경로, 실패 시 칩 폴백. 사용자 선호 충족하되 매핑 정확성은 LLM 강점(칩)에. 단일 실패점 제거.

## 결론
하이브리드 채택. 위계 명시 — 칩+드롭다운 주, 박스 탭 보조. 자동매핑(field 분류) LLM 강점이라 적극 활용. 재인식은 "재추첨" 성격을 UX에 반영. (전원 합의, 소수의견 없음)
