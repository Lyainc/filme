# UNRESOLVED — OCR 결과 검수/편집 모달 UX

토픽 4개는 전원 합의로 닫혔으나, 합의가 **측정·실측에 조건부**인 항목과 의도적으로 후속으로 미룬 항목을 기록한다.

## 조건부 합의 (해결됨 - 폐기)

### U1. 박스 탭 채택 여부 — 폐기됨
- 이전 합의(하이브리드에서 박스 탭을 보조로)는 박스 중심점 적중률 70% 이상을 전제했으나, 측정 결과 GPT-4o-mini의 bounding box 품질이 실사용 불가 수준으로 낮음이 확인됨.
- 이에 따라 bounding-box review UI(A1~A3)는 개발 취소(Unimplemented)되었으며, 현재는 Optimistic Injection 전략으로 변경됨.

### U2. 자동매핑 정확도 — 폐기됨
- 박스 매핑 자체가 무의미해짐에 따라, 필드 자동 분류 후 UI에서 박스를 통해 매핑을 수정하는 접근 역시 폐기됨.

## 후속 보류 (의도적 제외)

### U3. 핀치줌/팬 지원 여부 — 폐기됨
- Bounding-box review UI가 폐기되었으므로 모바일에서의 이미지 핀치줌/팬 요구사항도 함께 소멸함.

### U4. 재인식 비용·호출 제한 — 유지
- Optimistic Injection 후 재시도(Undo & Retry) 시마다 LLM을 다시 호출하게 됨.
- 기존 rate limit(시간당 10·일당 50)이 1차 방어막 역할을 하며, 별도 제한은 아직 두지 않음.
- **재검토 시점**: 실사용에서 재인식 남용 패턴이 보일 시.

## 폐기된 대안 (기록용)

- **순수 박스 탭(좌표 주 경로)**: 좌표·터치 두 리스크에 단일 종속 → 단일 실패점. 기각.
- **N×N 매핑 동시 시각화(색깔 선 연결)**: 인지부하 과다. 기각, "탭→팝오버 한 번에 하나" 채택.
