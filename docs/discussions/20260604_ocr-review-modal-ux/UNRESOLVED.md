# UNRESOLVED — OCR 결과 검수/편집 모달 UX

토픽 4개는 전원 합의로 닫혔으나, 합의가 **측정·실측에 조건부**인 항목과 의도적으로 후속으로 미룬 항목을 기록한다.

## 조건부 합의 (측정 대기)

### U1. 박스 탭 채택 여부 — #34 게이트 결과에 종속
- 현재 합의(하이브리드에서 박스 탭을 보조로)는 **박스 중심점 적중률 ≥70%**를 전제.
- 측정 결과 미달이면 합의가 자동으로 "순수 칩(박스 탭 폐기)"으로 이동.
- **미해결 이유**: 실제 `gpt-4o-mini` 좌표 품질을 아무도 측정한 바 없음. PoC A2 전까지 확정 불가.
- **해소 조건**: A1·A2 완료, #34에 적중률 코멘트.

### U2. 자동매핑 정확도 — "예외만 수정" 가정의 실측치
- TOPIC 2 결론("가치 희석은 자동매핑이 정확할수록 작아짐")은 **자동 field 분류가 충분히 맞다**는 가정에 의존.
- 분류 정확도가 낮으면 "예외만 수정"이 아니라 "전부 재매핑"이 되어 가치 희석 우려가 되살아남.
- **해소 조건**: A3 구현 후 동일 5~10장으로 field 분류 정확도도 함께 측정.

## 후속 보류 (의도적 제외)

### U3. 핀치줌/팬 지원 여부
- Frontend Engineer가 "모바일에서 작은 텍스트를 보려면 핀치줌이 필요할 수 있고, 그러면 좌표 변환이 4단으로 늘어 복잡도 급증"이라 지적.
- 하이브리드(칩 기반)에서는 박스 탭이 보조이므로 **줌 없이 시작**, 박스 탭 정밀도가 부족하면 칩으로 처리 가능 → 1차 PoC에서는 보류.
- **재검토 시점**: 박스 탭을 주 경로로 승격하려는 논의가 생길 때만.

### U4. 재인식 비용·호출 제한
- 재인식이 매번 LLM 호출(비용·rate limit)인데, 무제한 재시도 허용 여부 미결.
- 기존 rate limit(시간당 10·일당 50)이 1차 방어. PoC 단계에선 별도 제한 두지 않음.
- **재검토 시점**: 실사용에서 재인식 남용 패턴이 보이면.

## 폐기된 대안 (기록용)

- **순수 박스 탭(좌표 주 경로)**: 좌표·터치 두 리스크에 단일 종속 → 단일 실패점. 기각.
- **N×N 매핑 동시 시각화(색깔 선 연결)**: 인지부하 과다. 기각, "탭→팝오버 한 번에 하나" 채택.
